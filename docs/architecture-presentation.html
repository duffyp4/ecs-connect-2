Who Uses ECS Connect?

Four roles interact with the system at different stages of the job lifecycle.

💻
CSR

Customer Service Rep

Creates jobs, dispatches forms, manages the job lifecycle from their desktop.

  • Creates new jobs with customer & part details
  • Dispatches pickup forms to drivers
  • Checks in jobs and assigns technicians
  • Dispatches delivery forms when service is complete
  • Monitors job progress on the dashboard
🔧
Technician

Shop Technician

Performs emissions service on parts at the shop. Uses a phone or tablet.

  • Receives emissions service log on their device
  • Reviews pre-filled job and part information
  • Records pass/fail results for each part
  • Notes repairs performed and failure reasons
  • Submits completed form (works offline too)
🚚
Driver

Pickup & Delivery Driver

Picks up parts from customers and delivers completed parts back.

  • Sees assigned pickups and deliveries on their phone
  • Fills in item count and pickup notes
  • Records who received the delivery
  • GPS location captured automatically
  • Can submit forms even without cell service
🔐
Admin

Administrator

Manages users, monitors metrics, and oversees operations.

  • Manages the user whitelist and role assignments
  • Views operational dashboard and KPIs
  • Has full access to all CSR capabilities
  • Manages shop and technician configuration
  • Controls feature flags and system settings

Job Lifecycle

Follow a job from creation to delivery.

CSR

Job Created

CSR fills out customer info, adds parts, and selects arrival path (pickup, direct drop-off, or shipment).

CSR

Pickup Dispatched

A pickup form is created in the database and a WebSocket notification is sent to the driver's phone.

DRIVER

Pickup Completed

Driver opens the form on their phone, confirms item count, adds notes. GPS captured automatically. Job moves to "picked up".

CSR

Shop Check-In

CSR checks the job in at the shop, assigns a technician. An emissions service log form is dispatched to the tech.

TECHNICIAN

Service Performed

Tech opens the form, sees all pre-filled job and part info. For each part, records pass/fail, repairs, and ECS part number. Submits when done.

SYSTEM

Parts Updated

Server processes the submission: updates each part's pass/fail status, records repairs, transitions job to "service complete".

CSR

Delivery Dispatched

CSR dispatches a delivery form to a driver with delivery address and order numbers.

DRIVER

Delivery Completed

Driver delivers parts, records who received them, adds notes. Job moves to "delivered". Lifecycle complete.

What Changed

Replacing external dependencies with native capabilities.

Before (GoCanvas)

  • $49-79/user/month for GoCanvas licenses
  • Forms dispatched via GoCanvas API with field ID mapping
  • XML webhooks parsed with xml2js
  • Field map JSON files regenerated when forms change
  • Separate GoCanvas mobile app required
  • Round-trip: API dispatch → mobile app → XML webhook → parse
  • 2,900 lines of integration code
  • Replit hosting with cold starts
  • Replit OIDC auth with passport sessions

After (Native)

  • No per-user form fees
  • Forms are React components with type-safe validation
  • Direct database writes via REST API
  • Schema changes are code changes (Drizzle + Zod)
  • Same ECS Connect PWA for all roles
  • Direct: submit form → update database
  • ~3,500 lines of native form components
  • Railway with always-on hosting
  • Clerk auth with JWT tokens and user dashboard

Safe Migration

A feature flag lets you switch between GoCanvas and native forms instantly.

The Toggle

USE_NATIVE_FORMS=true

When set to true, all three form types (pickup, delivery, emissions) are dispatched through the native system. When false or unset, everything goes through GoCanvas as before. Flip it back instantly if anything goes wrong.

🚀

Phase 1: Test on Staging

Deploy to Railway with USE_NATIVE_FORMS=true. Test all three form types end-to-end while Replit production stays on GoCanvas.

🔄

Phase 2: Switch Production

Once validated, set USE_NATIVE_FORMS=true in production. GoCanvas webhook endpoint stays active as a safety net.

🗑

Phase 3: Clean Up

After running native forms for a week with no issues, remove ~2,900 lines of GoCanvas integration code and cancel GoCanvas licenses.
